HƯỚNG DẪN DEPLOY BACKEND NODE.JS LÊN RENDER (MIỄN PHÍ)
======================================================

Dự án của bạn là một Monorepo (chứa cả backend và frontend trong cùng 1 repo). Do đó, khi deploy lên Render, bạn cần cấu hình cẩn thận phần "Root Directory".

Dưới đây là các bước chi tiết để Deploy thư mục `backend` lên Render:

BƯỚC 1: KẾT NỐI GITHUB VỚI RENDER
-----------------------------------
1. Truy cập vào trang web: https://dashboard.render.com/
2. Đăng nhập bằng tài khoản GitHub của bạn (Sign in with GitHub).
3. Tại màn hình Dashboard chính, nhấn nút "New" (Mới) ở góc trên bên phải.
4. Chọn "Web Service" (Dịch vụ Web).
5. Chọn "Build and deploy from a Git repository" (Xây dựng và triển khai từ kho lưu trữ Git) rồi nhấn "Next".
6. Danh sách các kho lưu trữ (repository) trên GitHub của bạn sẽ hiện ra. Tìm kho lưu trữ `booking_system_deploy_test` và nhấn nút "Connect" (Kết nối).

BƯỚC 2: CẤU HÌNH DỊCH VỤ BACKEND
-----------------------------------
Bạn cần điền chính xác các thông tin sau để Render hiểu cách chạy thư mục backend của bạn:

- Name (Tên): Đặt tên tùy ý, ví dụ: `booking-system-backend-api`
- Region (Khu vực): Chọn `Singapore (SEAS)` để có tốc độ kết nối về Việt Nam nhanh nhất.
- Branch (Nhánh): `main`
- Root Directory (Thư mục gốc): Bắt buộc điền là `backend` (Vì mã nguồn Node.js của bạn nằm trong thư mục này, không phải thư mục ngoài cùng).
- Runtime (Môi trường chạy): `Node`
- Build Command (Lệnh cài đặt): `npm install`
- Start Command (Lệnh khởi động): `npm start` (Lệnh này Render sẽ gọi vào `node src/server.js` mà chúng ta đã cấu hình trong package.json).

- Instance Type (Loại máy chủ): Chọn gói `Free` (Miễn phí).

BƯỚC 3: CẤU HÌNH BIẾN MÔI TRƯỜNG (ENVIRONMENT VARIABLES)
-----------------------------------------------------------
Kéo xuống phần "Environment Variables" và nhấn "Add Environment Variable". 
Bạn phải copy thủ công từng dòng trong file `.env` ở máy tính lên Render (Render không tự đọc file .env trên GitHub vì file đó đã bị .gitignore chặn).

Bạn mở file `backend/.env` ra và copy paste đầy đủ các biến sau vào giao diện của Render:
(Lưu ý: Không copy dấu ngoặc kép " " nếu có)

1. Biến Hệ thống:
   - PORT: `5000`
   - NODE_ENV: `production`

2. Biến Database (Neon Postgres):
   - DATABASE_URL: (Copy link postgresql://... của bạn)

3. Biến Redis (Upstash):
   - REDIS_URL: (Copy link rediss://... của bạn)

4. Biến RabbitMQ (CloudAMQP):
   - RABBITMQ_URL: (Copy link amqps://... của bạn)

5. Biến OpenSearch/Elasticsearch (Searchly):
   - OPENSEARCH_NODE: (Copy link https://...searchly.com của bạn)
   - OPENSEARCH_API_KEY: (Mật khẩu hoặc key nếu có, nếu link URL đã chứa password thì không cần biến này)

6. Các biến JWT và Email (Nếu có trong .env):
   - JWT_SECRET: ...
   - JWT_REFRESH_SECRET: ...
   - EMAIL_USER: ...
   - EMAIL_PASSWORD: ...

Sau khi điền đủ tất cả các biến, nhấn nút "Create Web Service" (Tạo dịch vụ Web) ở dưới cùng.

BƯỚC 4: CHỜ ĐỢI VÀ LẤY LINK API
-----------------------------------
- Render sẽ bắt đầu quá trình Build (Tải các thư viện, cài đặt Node.js...). Bạn có thể theo dõi trong màn hình Logs đen đen.
- Quá trình này mất khoảng 2 - 5 phút.
- Khi màn hình Logs hiện ra dòng chữ `Your service is live 🎉`, nghĩa là Backend của bạn đã lên mạng thành công!
- Ở góc trên cùng bên trái (dưới tên dự án), bạn sẽ thấy một cái link có dạng: `https://booking-system-backend-api.onrender.com`. Đây chính là đường link API để Frontend của bạn gọi tới.

LƯU Ý QUAN TRỌNG KHI DÙNG GÓI FREE CỦA RENDER:
---------------------------------------------
1. Ngủ đông (Spin down): Nếu không có ai truy cập API trong vòng 15 phút, máy chủ sẽ tự tắt để tiết kiệm tài nguyên.
2. Thức dậy (Spin up): Khi có người truy cập lại, máy chủ sẽ mất khoảng 30-50 giây để khởi động lại (Cold start). Do đó, lượt request đầu tiên sẽ rất chậm. Các request sau đó sẽ nhanh bình thường.

Chúc bạn Deploy thành công! Mọi thứ đã được đóng gói hoàn hảo trên GitHub rồi.
